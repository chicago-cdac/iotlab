Welcome to the IoT Lab at CDAC/UChicago!

The IoT Lab is a place dedicated to studying computer security, privacy, and digital self-defense, but we are also open to serve as a testbed for hacking in general. We have awesome datasets you can use to get started with Data Science and Machine Learning.

1. Lab Policy

  • All the hardware present is the lab should not be taken or leave the room.
  • Please check-in / check-out the area you intend to work.
  • Please check-in / check-out the materials from the lockers.
  • Use the Green / Red signs to mark the status (Free / Busy) of the area you're using.
  • Do not leave any sensitive data in the computers' storage sdcards.
  • Bring your own sdcard / storage to save your data/work.
  • The sdcards are likely to be reflashed from time to time.
  • Do not run any experiment that can harm lab resources or someone else's work.
  • Please leave the area you checked-in the same way you found it.
  • The Lab is under is a heavy network monitoring / surveillance.
  • Please DO NOT install any malicious code at any time or at any circumstances.
  • Please DO NOT install or run security experiment without the consent of the lab's supervisor.
  • By check-in, you agree with this policy.

  • The Lab is composed of multiple connected IoT Devices.

Analyzing .pcap Data In this notebook, we're going to analyze the packet capture file .pcap using jupyter, pandas and bokeh. The packet capture was collected from the Samsung Smart Camera inside the lab. There's normal network traffic inside .pcap. During the packet capture, we established 2 video sessions in sequence giving a 2-minute-ish interval between the 2 sessions. We'll run cell-by-cell of the jupyter notebook from:
